## What Was Built

### Backend: validate-activation endpoint
New endpoint `GET /api/v1/projects/{id}/validate-activation`:
- Returns `ActivationValidationResponse` with `can_activate` boolean and `checks` array
- Each check has: id, category, severity, message, fix_action
- Categories: workflow, data_source, layouts, permissions
- Severities: blocker (prevents activation), warning, passed

Validation checks:
- Status is draft (blocker if not)
- Workflow configured (warning if not)
- Layout configured (blocker if not)
- Team assigned (warning if not)
- Data source configured (warning placeholder)

### useActivationValidation.ts
Hook for fetching and managing validation state:
- Fetches from validation endpoint
- Sorts checks by severity (blockers first)
- Groups checks by category
- Provides counts: blockerCount, warningCount, passedCount

### ChecklistItem.tsx
Individual check display:
- Severity icon: XCircle (red), AlertTriangle (yellow), CheckCircle (green)
- Message text
- "Fix" button with navigation for non-passed items

### ActivationChecklist.tsx
Modal dialog for activation:
- Header with project name
- Summary badges showing blocker/warning/passed counts
- Sections grouped by category
- Fix buttons navigate to relevant sections
- "Activate" disabled until blockers resolved

### ActivationSuccess.tsx
Success modal after activation:
- Celebratory green checkmark
- "Project Activated!" message
- Next steps guidance:
  1. Invite annotators
  2. Monitor dashboard
  3. Configure quality settings
- "Go to Dashboard" and "Stay Here" buttons
